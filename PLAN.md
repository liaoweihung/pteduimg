# 台灣局部眼用藥品資料庫與藥師版互動查詢網站計畫

## 目前狀態

- 已閱讀 `AGENTS.md`、專案結構與 Git 狀態。
- 已嘗試尋找可用的 Library / 共享 Sources 工具；目前工作環境未提供可搜尋 ChatGPT Library 的工具。
- 已依 fallback 檢查本機 `research/work_handoff/` 與專案內 `HANDOFF_MANIFEST.json`；目前未找到可驗證的交接 manifest。
- 在取得 `status: ready` 的 `HANDOFF_MANIFEST.json` 前，不使用未驗證資料替代 Work 交接包。

## 取得與驗證 Work 交接包

1. 搜尋 `眼用藥品資料庫交接包`。
2. 找出 `HANDOFF_MANIFEST.json` 中 `status` 為 `ready` 的最新版。
3. 依 `created_at` 與 `version` 判斷版本。
4. 將該版本 materialize 到 `research/work_handoff/<version>/`，避免覆蓋既有使用者檔案。
5. 依 manifest 驗證必要檔案、檔名、SHA-256、CSV 可讀性與 README 欄位規格。
6. 產出 `research/work_handoff/IMPORT_REPORT.md`。

## 資料處理

1. 建立可重複執行的處理程式，從交接 CSV 產生清理後資料庫與網站 JSON。
2. 將人工修正規則保存在可檢查的設定或對照表，不直接寫死於輸出 JSON。
3. 篩選台灣藥證局部眼用成品，排除原料藥、口服、注射、全身性、醫材與非藥證商品。
4. 建立物質角色分類：有效成分、防腐劑/保存劑、基劑或溶媒、緩衝劑、等張調節劑、黏度調節劑、螯合劑、界面活性劑、抗氧化劑、酸鹼調節劑、眼藥膏基劑、其他賦形劑、待確認。
5. 分別計算有效成分、保存劑與賦形劑統計，且所有統計可回查產品與藥證。
6. 產出 `DATA_AUDIT.md`，列出疑似混入非局部眼用產品、排除量、資料完整率與待人工確認項目。

## 網站

1. 保留現有靜態 GitHub Pages 架構，新增藥師版查詢頁與資料檔，不破壞既有卡片網站。
2. 提供四個主要入口：適應症、有效成分、藥品分類、商品或藥證搜尋。
3. 提供獨立製劑資訊入口：保存劑、無保存劑產品、賦形劑、製劑功能、劑型組成比較。
4. 支援手機優先卡片結果、桌面表格/雙欄、有效/失效藥證切換、明確狀態標籤與基本無障礙。
5. 顯示資料更新日期、資料來源與專業免責聲明。

## 驗證

1. 執行資料驗證與單元測試。
2. 執行生產建置。
3. 使用瀏覽器測試手機版搜尋、篩選、商品詳細頁、失效藥證勾選與製劑資訊功能。
4. 補充 README，說明資料更新、處理、網站啟動、建置與測試方法。
